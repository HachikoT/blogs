# 运算符优先级（由高到低）

|     类别     |                                                             运算符                                                             |  结合性  |
| :----------: | :----------------------------------------------------------------------------------------------------------------------------: | :------: |
|  单目运算符  | `++,--`（后缀自增，自减）、`+,-`（正负号）、`!`（逻辑非）、`^`（按位取反）、`*`（指针解引用）、`&`（取地址）、`<-`（通道接受） | 从右到左 |
| 乘法类运算符 |                 `*`（乘）、`/`（除）、`%`（取模）、`<<`（左移）、`>>`（右移）、`&`（按位与）、`&^`（按位清空）                 | 从左到右 |
| 加法类运算符 |                               `+`（加）、`-`（减）、`%`（取模）、`\|`（按位或）、`^`（按位异或）                               | 从左到右 |
|  比较运算符  |                   `==`（相等）、`!=`（不相等）、`<`（小于）、`<=`（小于等于）、`>`（大于）、`>=`（大于等于）                   | 从左到右 |
|  逻辑运算符  |                                                `&&`（逻辑与）、`\|\|`（逻辑或）                                                | 从左到右 |
|  赋值运算符  |                                            `=,+=,-=,*=,/=,%=,<<=,>>=,&=,\|=,^=,&^=`                                            | 从右到左 |

- **Go中没有前缀自增和自减，并且后缀自增和自减只能作为语句，不能作为表达式（`m = n++`会编译报错`expected ';', found '++'`）**
- **调整了运算符优先级，更符合人类的直觉逻辑了，例如在C语言里面`n << 1 + 2`等效于`n << (1 + 2)`，在Go里面就等效于`(n << 1) + 2`**
- **对有符号整型使用右移运算符执行的是算数右移，也就是按位右移，填充符号位**
- **按位取反运算符和按位异或运算符是一样的，都是`^`，当作为单目运算符时就代表是按位取反操作**
- **Go没有三元运算符（`?:`），因为Go的社区认为很多人使用三元运算符实现很长的判断语句，根本不利于代码的理解和阅读**
